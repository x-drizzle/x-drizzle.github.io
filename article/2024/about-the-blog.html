关于此博客

　　搭建一个属于自己的博客是我心水了很久的一件事儿，从2022年就已经开始尝试去了解搭建博客的相关知识，现如今也算是终于有了个能勉强满足我的基本需求的博客，虽然并不全是自己一行行敲出来的，但是这一路还是收获颇多。

　　可以访问到这篇博客的网址暂时有两个，分别是 https://no.6u.gs 和 https://xiix.in ，是利用 Hexo + Github Pages + Cloudflare Pages 以及本地部署搭建完成的静态网页博客，目前我认为算是比较折中的一种方案。

　　当然这并不是一开始我使用的域名。最开始的是一个与我姓名缩写有关系的域名（ yzow.cn ），那段时间发现很多大佬都有很好记的域名作为博客地址，奈何我没那精力与财力。后边了解到各大云服务商有个功能叫“一口价域名”，运气好能遇到一些不太昂贵的好记域名，然后某一天在阿里云的一口价域名页面看到了上述的域名，只有四个字母，且还是咱们国家的顶级域名 .cn ，于是并未犹豫便拿下了。

　　买了域名的下一步是什么呢，当时其实有接触到 GitHub Pages ，但是无奈它只能托管静态网页，但是我这么喜欢折腾，总觉得静态网页有点没意思，便想着去哪整个服务器玩玩，逛了不少论坛发现好多人推荐练手建议从免费VPS入手，既不用去了解Linux的知识，也不用愁搭建环境时遇上的种种报错，当然也免去了被DDos攻击以及cdn防护等的相关事宜。

　　VPS似乎有不少云服务商提供免费的，但是我所能接触到最多的大多都是国内的，一言不合就让你实名认证，一实名那些什么都会有限制，跟写公众号和微博没啥区别，而且大部分都是安装好了管理面板，简单是简单，感觉太简单了点。于是我又开始找寻位于城墙外的VPS，找寻许久发现大部分都会有每月一美刀的机型，而且很少有装好管理面板，基本都支持SSH，既然是学习知识交点学费实践实践也不亏，于是便开始了Linux的学习。

　　到后来又发现一个问题，这些VPS都是共享的IP地址，只要共享IP地址的这么多人中的有一个人去干坏事，我部署的网页也会跟着受影响。于是便又萌生了买服务器的想法，但是服务器又涉及到IP地址，IPv4地址早在2019年11月26日就耗尽了，说白了也就是带IPv4的地址的服务器不会太便宜，IPv6的地址一大长串又不好记。

　　但是也不是没有难题。咱们一直以来用的都是可视化的windows系统，对于服务器上Linux系统，首先它并不是可视化界面（可以安装但是通常没必要），几乎用不上鼠标操作，其次其需要了解掌握大量的指令，此外不同Linux系统的指令都有些许差异，远程连接还需要了解SSH、公钥私钥的概念，不同SSH工具之间也有差异……
